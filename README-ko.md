**실증 연구 전문가를 위한 에이전트 스킬 배포판.** 마케팅용 목록이 아닙니다 — 본 저장소에 **1,145개의 스킬을 직접 벤더링하고 카탈로그화**했으며, **수치 벤치마크, eval 하니스, 보안 감사, CI**로 감쌌고, 여기에 더 넓은 생태계의 **119개 저장소에 걸친 23,000개 이상의 스킬**을 정리한 큐레이션 지도를 더했습니다.

AERS는 두 가지를 동시에 제공합니다: (1) 완전한 실증 파이프라인 — 데이터 정제 → 식별 → 추정 → 강건성 → 표/그림 → 투고 가능한 초안 — 을 실행하는 소수의 **자체 제작 플래그십 스킬**, 그리고 (2) 연구 워크플로 단계별로 정리한, **큐레이션되고 보안을 고려한** 실증 연구 스킬 생태계 카탈로그입니다. 차별점은 개수가 아니라, 플래그십의 동작이 단지 주장에 그치지 않고 **알려진 정답에 대해 검증된다**는 점입니다.

## 2분 만에 직접 검증하기

여기서 가장 설득력 있는 것은 숫자가 아니라, 플래그십 파이프라인의 동작을 **API 키나 유료 모델 없이도 확인할 수 있다**는 점입니다. Python 3만 있으면 됩니다:

```bash
git clone https://github.com/brycewang-stanford/Auto-Empirical-Research-Skills.git
cd Auto-Empirical-Research-Skills
make check        # repo validation + unit tests + eval lint + numeric benchmark
```

벤치마크가 가장 설득력 있는 부분입니다: **매 실행마다 원본 데이터셋에서 gold 답을 재계산**하므로, 숫자를 하드코딩해 점수 통과를 위조할 수 없습니다. 기본 상태로 다음을 복원합니다:

- **LaLonde (1986) / Dehejia–Wahba (1999)** — 순진한 관찰적 비교는 *부호가 틀린* 값(−$635)을 내놓지만, 공변량 조정이 이를 양수(≈ +$1,548)로 뒤집어 실험 벤치마크(≈ +$1,794)에 가까워집니다.
- **Card (1995)** — 교육 수익률에 대한 IV(0.131)가 OLS(0.075)를 *상회*하며, 1단계 F(13.3)는 숨기지 않고 보고됩니다.
- 여기에 staggered-DID(TWFE 편향 vs. group-time 진실), 샤프 **RDD**, 그리고 **나쁜 통제 / 처치 후 편향** 함정까지 포함됩니다.

파이프라인은 **함정을 드러내고, 오해를 부르는 숫자를 헤드라인으로 내세우지 않으며, 재계산된 진실과 일치할 때**에만 통과합니다. [`benchmark/`](benchmark/)와 [`docs/TRUST.md`](docs/TRUST.md)의 전체 신뢰 개요를 참고하세요.

> 💡 **호스팅된 엔드투엔드 버전을 원하시나요?** 조립을 건너뛰세요 — [**copaper.ai**](https://copaper.ai)가 실증 파이프라인을 대신 실행해 드립니다. 이 카탈로그를 만든 동일한 스탠퍼드 방법론 팀이 함께 구축했습니다.

## 플래그십 파이프라인 스킬

**동일한 8단계 실증 루프** — *데이터 정제 → 변수 구성 → 기술통계 → 진단 → 추정 → 강건성 → 메커니즘/이질성 → 발표 수준 표 & 그림* — 의 네 가지 병렬 구현에, 투고 및 탈AIGC 스택을 더했습니다. 각각은 **점진적 공개(progressive disclosure)**를 사용합니다: `SKILL.md`에는 얇은 정규 호출 골격만 두고, 단계별 심층 레퍼런스 매뉴얼은 필요할 때만 로드합니다. 이들은 공존하므로, 스택과 사용 사례에 따라 선택하세요.

## 23K 스킬 더미 그 이상인 이유

공개 스킬 개수는 부풀리기 쉽고, 최근 연구들은 대규모 스킬 인덱스가 흔히 중복적이며 때로는 안전하지 않다는 것을 보여줍니다. AERS는 단순 개수가 아니라 **검증 가능한 품질**로 경쟁합니다. 아래의 모든 층위는 `make check`로 로컬에서, 그리고 CI에서 실행됩니다.

| 층위 | 무엇을 잡아내는가 | 위치 |
|---|---|---|
| **수치 벤치마크** | 실제 데이터에서 재계산한 진실과 일치하지 않는 보고 숫자 — 순진한 DID 부호 함정, 1단계 F 없는 약한 IV, staggered 타이밍 하의 TWFE 편향, RDD 추세 교란, 처치 후 나쁜 통제 | [`benchmark/`](benchmark/) · 5개 과제 |
| **Eval 하니스** | 산문 수준 실패: 약한 IV의 거짓 안심, staggered-DID의 TWFE 오용, 조작된 인용, 안전하지 않은 `curl \| bash` 설치, 다중 검정 남용, AER 준수 누락 | [`eval-harness/`](eval-harness/) · 17개 시나리오 / 95개 루브릭 항목 |
| **보안 감사** | pipe-to-shell, 리버스 셸, 자격증명 유출, 프롬프트 인젝션을 13개 위험 카테고리에 걸쳐 점검 — 6단계, 40개 이상의 hook 스크립트를 손으로 검토 | [`SECURITY-SCAN-REPORT.md`](SECURITY-SCAN-REPORT.md) |
| **출처 & 라이선스** | 벤더링되지 않은 출처, 라이선스 위험, 1,145개 카탈로그 스킬 전반의 위생 드리프트 | [`docs/LICENSE_AUDIT.md`](docs/LICENSE_AUDIT.md) · [`docs/SKILL_QUALITY.md`](docs/SKILL_QUALITY.md) |
| **CI & 호환성** | 카탈로그 신선도, 깨진 로컬 링크, GitHub Actions 정책, Python 3.9 **및** 3.12 문법 하한선 | [`.github/workflows/`](.github/workflows/) · 6개 워크플로 |

```bash
make catalog     # regenerate catalog, provenance, audit, enrichment
make validate    # freshness + link / frontmatter checks
make check       # full gate: validate + Python compile + unit tests + eval lint + benchmark
```

이 신뢰 표면은 **필요조건이지 충분조건은 아닙니다** — 정규식 루브릭은 산문을 보증하지 못하고, 작은 벤치마크는 모든 설계를 다루지 못합니다. 이는 *알려진 고비용 실수에 대해 빠르게 실패하도록* 설계되었습니다. 정직한 적용 범위는 [`docs/TRUST.md`](docs/TRUST.md)와 [`docs/QUALITY_GATE.md`](docs/QUALITY_GATE.md)에서 확인하세요.

## 보안

**원본 52개 스킬 컬렉션 / 2,940개 이상의 파일**이 체계적 감사를 통과했습니다 — **52/52 CLEAN, FLAGGED 0건**: 악성 프롬프트, 바이러스, 리버스 셸, 프롬프트 인젝션 없음. 모든 "민감한" 적중은 세 가지 정당한 카테고리 중 하나로 검증되었습니다: **방어적 보안 규칙**, **정당한 학술 API 호출**(arXiv / CrossRef / PubMed / FRED / World Bank / OECD / BLS), 또는 **표준 Claude Code 워크플로 hook**(모두 로컬 파일 작업, 네트워크 IO 없음).

![Skills 보안 스캔 개요](images/security-scan/security-scan-01-overview.png)

6단계, 심층 방어(defense-in-depth): **13개 위험 카테고리**에 걸친 자동 grep → **hook을 포함한 6개 스킬과 그 40개 이상의 hook 스크립트** 전부 수동 검토 (어디에도 `Bash(*)` 와일드카드 없음) → 3개의 병렬 에이전트 콘텐츠 감사 → 보충 무결성 검사 (숨겨진 Unicode, 인코딩 이상, HTML 인젝션, 네트워크 import).

> **핵심 통찰:** 가장 크다 ≠ 가장 위험하다. 가장 큰 스킬들이 모두 통과했으며, [17-DAAF](skills/17-DAAF-Contribution-Community-daaf/)는 오히려 보안을 의식한 설계의 기준을 세웁니다 (14개 방어 hook + 32개 deny 규칙 + 능동적 자격증명 스캔).

베이스라인 이후 새로 벤더링된 추가분은 [`catalog/provenance.json`](catalog/provenance.json)과 [`docs/SKILL_AUDIT.md`](docs/SKILL_AUDIT.md)에서 추적됩니다 — `make audit`를 실행하세요. 전체 보고서: [**SECURITY-SCAN-REPORT.md**](SECURITY-SCAN-REPORT.md).
