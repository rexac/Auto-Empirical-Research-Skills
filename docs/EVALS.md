## AER Staggered DID Identification Audit

- ID: `aer-identification-staggered-did-audit`
- Category: `identification_review`
- Skill: [`skills/50-brycewang-aer-skills/skills/aer-identification/SKILL.md`](../skills/50-brycewang-aer-skills/skills/aer-identification/SKILL.md)

Prompt:

> Audit a draft paper that estimates a staggered-adoption difference-in-differences model with heterogeneous treatment effects and only a short pre-period. Produce an estimand-first diagnosis, threat matrix, required diagnostics, preferred modern DID estimators, and referee-facing language for why the design is or is not credible.

Expected artifacts:
- `estimand_diagnosis.md`
- `identification_threat_matrix.md`
- `required_diagnostics.md`
- `recommended_estimators.md`
- `referee_response_language.md`

Review checks:
- Separates the target estimand from the estimator.
- Flags TWFE contamination risks under staggered adoption and heterogeneous effects.
- Specifies concrete diagnostics rather than generic robustness claims.
- States what evidence would make the design non-credible.

Failure modes:
- Recommends TWFE as sufficient without qualification.
- Treats event-study pre-trends as definitive proof.
- Adds robustness checks without addressing the estimand.

## AER Replication Package Audit

- ID: `aer-replication-package-audit`
- Category: `replication`
- Skill: [`skills/50-brycewang-aer-skills/skills/aer-replication/SKILL.md`](../skills/50-brycewang-aer-skills/skills/aer-replication/SKILL.md)

Prompt:

> Audit a replication package for an accepted applied economics paper before AEA Data Editor review. Check README completeness, master script execution order, environment capture, raw versus derived data separation, restricted-data handling, random seeds, expected runtime, and table/figure reproduction mapping.

Expected artifacts:
- `replication_readme_review.md`
- `file_manifest.csv`
- `execution_order.md`
- `restricted_data_notes.md`
- `reproduction_mapping.md`

Review checks:
- Maps each paper table and figure to exact scripts and output files.
- Distinguishes public, restricted, raw, intermediate, and final artifacts.
- Captures runtime, software versions, seeds, and operating-system assumptions.
- Flags any steps that require human judgment or unavailable data.

Failure modes:
- Treats a code dump as a replication package.
- Omits data-use restrictions or access instructions.
- Does not define a single master execution path.
